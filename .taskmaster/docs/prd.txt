# Pomobox PRD: 타이머 일시정지/재개 (통계 정책 유지)

## 배경
Pomobox는 포모도로 타이머 앱이다. 현재 세션 진행 중 일시정지가 없어, 사용자가 잠깐 중단하면 흐름이 깨지거나 리셋을 강요받는다.

## 목표
- 진행 중인 Focus/Break 세션을 Pause/Resume 할 수 있게 한다.
- 기존 통계 정책(세션 증가/완료 세션 증가/누적 포커스 시간 계산 규칙)을 깨지지 않게 유지한다.

## 범위(In Scope)
- Focus/Short Break/Long Break 상태에서 Pause 버튼 노출
- Pause 시: 카운트다운 정지(시간 감소 없음), UI에 “일시정지” 상태 표시
- Resume 시: 남은 시간부터 재개
- Skip/Reset 동작은 기존 정책 유지(통계 변화 규칙 변경 금지)
- 상태 전환(포커스→브레이크, 브레이크→포커스) 로직이 Pause 상태에서도 안전하게 동작

## 비범위(Out of Scope)
- 멀티 디바이스 동기화
- 서버 저장/로그인 기능 추가
- 알림 시스템 전면 개편

## 수용 기준(Acceptance Criteria)
1) Focus 중 Pause를 누르면 시간이 더 이상 감소하지 않는다.
2) Resume을 누르면 Pause 직전 남은 시간에서 다시 감소한다.
3) Pause 상태에서 페이지 새로고침을 해도(기존 저장 정책이 있다면) 상태가 일관되게 복구된다.
4) Pause/Resume을 여러 번 반복해도 통계(세션/완료 세션/누적 포커스 시간)가 기존 정의대로 유지된다.
5) 빌드/린트/테스트(있는 경우)가 통과한다.

## 엣지 케이스/예외
- 00:00 도달 직전 Pause/Resume 연타 시에도 완료 처리 중복이 발생하면 안 된다.
- Pause 중 Skip/Reset을 누르면 기존 정책대로 즉시 전환/초기화된다.
